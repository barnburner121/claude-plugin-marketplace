# ClaudePlugins - Plugin Factory

Automated system for researching, generating, and publishing monetizable Claude Code plugins.

## Quick Start
- Run `/make-plugin` to generate 10 plugins with market research, code, and monetization strategies
- Generated plugins go to `generated-plugins/`
- Publishing scripts in `scripts/`

## Project Structure
- `.claude/skills/make-plugin/` — The main skill that drives the factory
- `templates/` — Base templates for MCP servers and plugins
- `scripts/` — Publishing, hosting, and marketplace setup scripts
- `generated-plugins/` — Output directory for generated plugins
- `.claude-plugin/` — This project's own plugin manifest + marketplace config
- `infrastructure/` — Docker/hosting setup (generated by setup-hosting.sh)

## Monetization Strategy
- Plugins are free on the marketplace for visibility
- MCP servers behind plugins use freemium model (free tier + paid Pro/Enterprise)
- Revenue via: MCPize (85/15 split), self-hosted (Stripe), npm distribution
- Target: hosted MCP SaaS with subscription billing

## Environment Variables Needed
- `STRIPE_API_KEY` — For payment processing on hosted servers
- `GITHUB_TOKEN` — For marketplace repo creation and npm publishing
- `NPM_TOKEN` — For publishing MCP server packages
- `MCPIZE_API_KEY` — For MCPize marketplace submissions
